## What This Package Contains

TT-Production is a complete self-hosted Docker Compose infrastructure stack for commercial
handoff and operator-managed deployment. It ships as a ZIP archive containing:

- **tt-core**: Core stack - PostgreSQL, Redis, n8n (queue mode), pgAdmin, RedisInsight
- **tt-supabase**: Optional self-hosted Supabase platform
- **Cloudflare Tunnel**: Optional zero-inbound-port public exposure
- **Addon system**: Optional services such as Metabase, WordPress, Kanboard, Qdrant, Ollama, and more
- **Documentation**: install, recovery, system requirements, and customer acceptance materials
- **Release evidence**: 100-check release validator, smoke framework, and audit summary

## Quick Start

### Linux (Recommended Production Path)

```bash
cd tt-core
bash scripts-linux/init.sh
bash scripts-linux/preflight-check.sh
bash scripts-linux/start-core.sh
bash scripts-linux/smoke-test.sh
```

### Windows (Development/Staging)

```powershell
cd tt-core
.\scripts\Init-TTCore.ps1
.\scripts\Preflight-Check.ps1
```

## Support Boundaries

This handoff package includes:

- Infrastructure stack source and configuration
- Documentation and customer/operator checklists
- Backup, restore, and support-bundle tooling
- Release validation and operational smoke testing

This handoff package does **not** include:

- Managed hosting or cloud infrastructure
- Ongoing maintenance or updates
- Custom application development
- Third-party service accounts such as Cloudflare or SMTP

---

## Operator Responsibilities

The receiving operator is responsible for:

1. Providing the target host or server infrastructure.
2. Reviewing and adjusting the runtime env values generated by init.
3. Configuring Cloudflare Tunnel token if public access is required.
4. Running `init.sh` / `Init-TTCore.ps1` to generate cryptographic secrets.
5. Completing the acceptance checklist before production use.
6. Running regular backups and periodic restore verification.
