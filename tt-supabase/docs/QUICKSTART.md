# TT-Supabase — Quick Start Guide (TT-Production v14.0)

## What is TT-Supabase?

TT-Supabase is an optional self-hosted Supabase BaaS stack bundled with TT-Production.
It runs all 13 Supabase services (Auth, REST API, Realtime, Storage, Edge Functions, Studio, etc.)
via Docker Compose with Kong as the single API gateway.

---

## URL Variables — Critical

Three URL environment variables serve different purposes.
Mixing them up **breaks OAuth redirects and auth confirmation emails**.

| Variable | Purpose | Example |
|---|---|---|
| `SUPABASE_PUBLIC_URL` | The public URL clients use to reach Kong (via Cloudflare Tunnel or reverse proxy). Set this to your Supabase subdomain. | `https://supabase.yourdomain.com` |
| `API_EXTERNAL_URL` | The base URL GoTrue (auth) uses when generating links in emails (invite, password reset, email confirmation). Usually **same** as `SUPABASE_PUBLIC_URL`. | `https://supabase.yourdomain.com` |
| `SITE_URL` | Your **frontend application** URL. GoTrue uses this for OAuth callback redirects. Must match what your frontend app uses. | `https://app.yourdomain.com` |

> If all three point to the same domain (e.g. your Supabase stack IS your app), set all three to the same value.

## Setup Steps

### 1. Initialize secrets

```bash
bash scripts-linux/init.sh
```

This generates: `POSTGRES_PASSWORD`, `JWT_SECRET`, `ANON_KEY`, `SERVICE_ROLE_KEY`, `LOGFLARE_API_KEY`,
`REALTIME_SECRET_KEY_BASE`, `REALTIME_DB_ENC_KEY`, `SUPAVISOR_SECRET_KEY_BASE`, `SUPAVISOR_VAULT_ENC_KEY`.

### 2. Configure your environment

Edit `compose/tt-supabase/.env` (generated by `init.sh` from the template):

```bash
# Required: set your real URLs (see table above)
SUPABASE_PUBLIC_URL=https://supabase.yourdomain.com
API_EXTERNAL_URL=https://supabase.yourdomain.com
SITE_URL=https://app.yourdomain.com

# Required: set a non-default admin username
DASHBOARD_USERNAME=your_admin_username

# Optional: configure SMTP for production email delivery
SMTP_HOST=smtp.resend.com
SMTP_USER=resend
SMTP_PASS=re_your_api_key
SMTP_PORT=587
```

### 3. Validate configuration

```bash
bash scripts-linux/preflight.sh
```

Fix any `[FAIL]` items before proceeding. Review `[WARN]` items carefully.

### 4. Start the stack

```bash
bash scripts-linux/start.sh
```

### 5. Verify

```bash
bash scripts-linux/smoke-test.sh
```

All checks should show `[OK]`.

## Access

| Service | URL |
|---|---|
| Studio Dashboard | `http://127.0.0.1:18000/project/default` — login with DASHBOARD_USERNAME/PASSWORD |
| REST API | `https://your.domain/rest/v1/` — requires `apikey` header |
| Auth API | `https://your.domain/auth/v1/` |

## PostgreSQL Version Note

TT-Supabase uses `supabase/postgres:15.8.1.085` (PostgreSQL 15).
TT-Core uses `postgres:16.6-alpine` (PostgreSQL 16).
These are **completely separate databases** on separate Docker networks.
The version difference is **intentional** — Supabase's image bundles required extensions
(pg_cron, pg_graphql, pg_net, pgmq, etc.) that are not available in the standard postgres image.

## Security Defaults

| Setting | Default | Notes |
|---|---|---|
| `DISABLE_SIGNUP` | `true` | Only admin-invited users can register |
| `FUNCTIONS_VERIFY_JWT` | `true` | All Edge Functions require auth |
| `DASHBOARD_USERNAME` | `__SET_ADMIN_USERNAME__` | **Must be set before starting** |
| Kong ports | `127.0.0.1` only | Requires Cloudflare Tunnel for internet access |

## Optional: Supavisor Connection Pooler

Supavisor is shipped but disabled by default. Enable it when you need direct
PostgreSQL access from many concurrent clients (e.g. connecting your own backend).

```bash
cd compose/tt-supabase
docker compose --profile optional up -d supavisor
```

Connection string: `postgresql://postgres:PASSWORD@127.0.0.1:16543/postgres`
